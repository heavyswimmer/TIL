# 데이터 타입

자바스크립트의 모든 값은 데이터 타입을 갖는데, 데이터 타입을 사용하는 이유는 크게 두 가지이다. 

> 1. 저장할 때 메모리에 확보해야 할 공간의 크기를 효율적으로 정하기 위해
> 2. 읽어들일 때 메모리에 2진수의 나열로 저장된 값을 어떻게 해석할 지를 판단하기 위해

데이터 타입은 크게 원시 타입과 객체 타입으로 분류된다. (객체 타입에 대한 설명은 따로 분류하여 작성함)

## 원시 타입

### 1. 숫자 타입
자바스크립트는 하나의 숫자 타입만 존재하며, 정수와 실수 구분 없이 모든 수를 **실수**로 처리한다.

### 2. 문자열 타입
토큰과 구분하기 위해 *작은따옴표*('') 안에 텍스트를 입력해서 사용한다.

### 3. boolean 타입
참과 거짓만을 나타내기 때문에 조건에 따라 의사를 결정하는 **조건문**에서 자주 사용된다.

### 4. undefined 타입
변수 선언 후 값을 할당하지 않은 상태일 때, 쓰레기 값이 참조되는 것을 방지하기 위해 `undefined` 값을 부여해 암묵적으로 **초기화**시키는 용도로 사용한다.

### 5. null 타입
개발자가 **의도적으로** 변수에 값이 없다는 것을 나타내고 싶을 때는 `undefined` 가 아니라 `null` 을 사용해야 한다.<br>
`null` 은 이전에 참조하던 값을 무효화시키고 참조된 값이 없는 상태로 만든다.

### 6. symbol 타입
심볼 값은 중복되지 않기 때문에 프로퍼티의 키를 만드는 데 사용된다.<br>
리터럴로 생성되는 다른 원시 타입들과는 다르게 `Symbol` 함수로 생성해야 한다.
